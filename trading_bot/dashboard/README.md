# Trading Bot Dashboard

A real-time dashboard for monitoring and controlling the trading bot strategies and performance.

## Features

- **Real-time Performance Monitoring**: Track portfolio performance, drawdowns, and volatility
- **Strategy Analysis**: View strategy weights, performance metrics, and correlations
- **Signal and Position Tracking**: Monitor current trading signals and open positions
- **Risk Management**: Keep track of risk alerts and market regime changes
- **Interactive UI**: Responsive design with interactive charts and tables

## Setup and Installation

1. Install required packages:
   ```
   pip install flask flask-socketio pandas numpy
   ```

2. Run the dashboard:
   ```
   python run_dashboard.py
   ```

3. Open your browser and navigate to:
   ```
   http://localhost:5000
   ```

## Dashboard Components

### Navbar
The top navigation bar provides access to different sections of the application and displays the connection status.

### Key Metrics
- **Current P&L**: Current profit and loss for the account
- **Sharpe Ratio**: Risk-adjusted return metric
- **Drawdown**: Maximum percentage drop from peak
- **Volatility**: Annualized standard deviation of returns

### Charts
- **Performance Chart**: Line chart showing portfolio value compared to benchmark
- **Strategy Weights**: Doughnut chart displaying allocation percentages across strategies
- **Correlation Chart**: Heatmap showing the correlation between different strategies

### Tables
- **Current Signals**: Latest trading signals generated by the strategies
- **Current Positions**: Open positions with entry price, current price, and P&L
- **Strategy Performance**: Performance metrics for each individual strategy

### Risk Monitor
- **Alerts**: Important notifications about market conditions or portfolio risks
- **Correlation**: Visual representation of diversification across strategies

## Architecture

The dashboard consists of:

1. **Flask Backend**: Provides API endpoints and WebSocket communication
2. **WebSocket Connection**: Enables real-time data updates
3. **Frontend**: HTML/CSS/JavaScript with Chart.js for visualizations

## Development

### Directory Structure
```
dashboard/
├── app.py                  # Flask application
├── run_dashboard.py        # Dashboard runner script
├── static/                 # Static assets
│   ├── css/
│   │   └── dashboard.css   # Dashboard styles
│   └── js/
│       └── dashboard.js    # Dashboard functionality
└── templates/              # HTML templates
    └── dashboard.html      # Main dashboard template
```

### Extending the Dashboard

To add new features to the dashboard:

1. Add new API endpoints in `app.py`
2. Update the HTML template in `templates/dashboard.html`
3. Add styling in `static/css/dashboard.css`
4. Implement JavaScript functionality in `static/js/dashboard.js`

## Connecting to Live Trading

By default, the dashboard uses simulated data. To connect it to your live trading system:

1. Replace the `TradingDataSimulator` class in `app.py` with your actual data source
2. Ensure your data follows the same structure or update the frontend accordingly
3. Implement proper authentication if needed
